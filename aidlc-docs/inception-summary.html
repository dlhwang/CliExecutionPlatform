LLM 기반 Workspace CLI 실행 플랫폼

사용자의 자연어 요청에 따라 안전하게 격리된 환경에서 파일 생성 및 CLI 도구를 실행하고, 실시간 과정과 결과를 SSE 스트리밍으로 전달하는 지능형 원격 설계 플랫폼 프로젝트 요약서

💡제품 컨셉 & 핵심 가치

본 플랫폼은 "자연어로 묘사하여 하드웨어/디자인 결과물 얻기"를 실현합니다. 사용자가 원하는 요구사항을 자연어로 작성하면, 시스템이 안전하게 격리된 가상의 작업 공간(Workspace) 내에서 코드를 수정하고 외부 설계 CLI 도구를 구동하여 3D 설계 파일과 이미지 형상을 실시간 제공합니다.

🎯첫 번째 적용 대상 및 예시 시나리오

MVP 단계의 핵심 타겟 도구는 오픈소스 3D CAD 스크립트 도구인 OpenSCAD CLI입니다. 사용자가 형태를 묘사하면 시스템은 백그라운드에서 OpenSCAD 코드를 빌드하여 즉시 출력용 STL 파일과 뷰어용 PNG를 제작합니다.

샤오미 워치 S4 충전 도킹스테이션 설계 시나리오

  • 자연어 인풋: "샤오미 워치 S4용 가로 거치형 충전 도킹스테이션 설계도를 만들어줘. 원형 충전기가 들어갈 홈도 뚫어줘."
  • 시스템 동작: 고유 Job 생성 및 디렉토리 할당 -> LLM이 설계 요구를 분석하여 README.md, design-spec.md, model.scad 생성 플랜 수립 -> 보안성 검사 후 파일 생성 -> OpenSCAD CLI 실행하여 preview.png 렌더링 및 output.stl 생성 -> SSE로 로그 중계 -> 최종 다운로드 제공.
  • 대화형 피드백: 생성된 이미지를 보고 "오른쪽 벽면에 케이블 통과 홈을 5mm 지름으로 뚫어줘"라고 요청하면 이전 model.scad를 자동 복제/수정하여 렌더링 재수행.

🎯 목표 및 동기

시중이나 인터넷 사이트에서 자신이 사용하는 기기(스마트폰, 스마트워치 등)의 정확한 치수 규격에 딱 맞는 도킹 스테이션을 찾기 힘듭니다. 직접 3D 설계를 하고 싶으나 전문 CAD 툴은 배우기 너무 어렵습니다. 코드를 몰라도 자연어로 말만 하면 치수가 들어간 완성도 있는 3D 출력용 STL 파일을 신속하게 얻고 싶습니다.

⚠️ 핵심 Pain Points (애로사항)

  • 기존 CAD 툴의 높은 진입장벽: 마우스 드래그 앤 드롭조차 어렵고, OpenSCAD 같은 코드 기반 툴은 아예 포기하게 됨.
  • 불투명한 설계 작업 과정: 원격 서버에 작업을 지시했을 때, 정상 렌더링 중인지 응답이 없는지 알 수 없어 불안함.
  • 네트워크 불안정으로 인한 데이터 유실: 모니터링 화면을 잠시 새로고침하거나 기차 안에서 네트워크가 일시 끊겼을 때 작업 로그 전체가 유실되어 복구 불가능함.
  • 치수 수정의 난해함: 다 만들어진 모델에서 충전 홈의 직경만 2mm 키우고 싶은데, 코드나 마우스를 만지기 두려워 처음부터 생성해야 함.

✨ 민수에게 제공하는 비즈니스 해결책

  • 자연어 인터페이스: 묘사만 하면 3D 코드를 대신 코딩 및 렌더링.
  • 실시간 진행 중계 (SSE): 현재 작업 상태와 CLI 빌드 출력을 끊김 없이 뷰어 제공.
  • 무중단 스트림 자동 복구 (Last-Event-ID): 새로고침하거나 유실되어도 직전 로그부터 자동 재동기화 및 이어붙임.
  • 점진적 대화 수정: "케이블 통로 추가해줘" 등의 추가 문장에 대해 이전 model.scad를 기반으로 똑똑하게 누적 수정 진행.

🗺️핵심 사용자 여정 (User Journeys)

김민수 씨가 제품을 처음 사용하여 원하는 출력물을 안전하게 획득하기까지의 순차적 흐름과 그 인수 조건(Acceptance Criteria)입니다.

1. 자연어 기반 설계 Job 요청

사용자가 "샤오미 워치 S4 충전 도크"를 요구사항으로 작성해 제출하면, 시스템은 안전하게 격리된 작업 영역을 즉시 확보하고 티켓을 발급합니다.

인수 요건 (Acceptance Criteria)
  • 요청 시 즉시 고유한 Job ID가 발급되며 데이터베이스에 등록된다.
  • 이 Job만을 위한 격리 디렉토리(Workspace)가 호스트 파일 서버에 생성되어 타인의 작업과 분리된다.

2. 실시간 진행 상황 및 로그 모니터링

설계가 렌더링되는 실시간 빌드 로그와 LLM 설계 상태가 SSE 스트리밍을 통해 사용자 화면에 실시간으로 흐르게 됩니다.

인수 요건 (Acceptance Criteria)
  • 사용자는 스트림 연결 `/api/v1/jobs/{job_id}/stream`을 통해 CLI의 빌드 출력을 실시간 관람할 수 있다.
  • 발행되는 모든 로그는 DB의 `event_logs` 테이블에 보존된다.

3. SSE 스트림 무중단 자동 복구 (Last-Event-ID)

연결이 끊기거나 사용자가 새로고침을 하더라도, 중복 로그 수신이나 누락 없이 마지막으로 읽었던 로그 다음부터 이어서 로그를 채워 넣습니다.

인수 요건 (Acceptance Criteria)
  • 클라이언트가 헤더에 `Last-Event-ID`를 싣고 재접속하면, 서버는 DB에서 해당 시점 이후에 쌓인 이벤트만 역정렬하여 먼저 일괄 재전송한다.
  • 재전송 완료 후 실시간 로그로 이음새 없이 자연스럽게 이어진다.

4. 아티팩트(3D 이미지 & STL) 미리보기 및 다운로드

설계가 완료되면 최종 생성된 이미지를 통해 3D 결과물을 미리 눈으로 확인하고, 만족 시 3D 프린터용 STL 파일과 3D 코드를 내려받습니다.

인수 요건 (Acceptance Criteria)
  • 성공적으로 렌더링된 preview.png와 output.stl의 다운로드 안전 Endpoint가 제공된다.
  • 경로 조작 등으로 호스트 컴퓨터 내의 시스템 주요 파일(예: OS 환경파일 등)을 요청 시 단호히 403 에러로 거절한다.

5. 대화형 피드백을 통한 설계 점진적 반복 수정

생성된 형상 이미지를 확인하고, 사용자가 "둥근 홈의 크기를 5mm 늘려줘"와 같이 피드백을 주면 시스템은 기존 코드를 똑똑하게 이어받아 부분 수정 후 재렌더링합니다.

인수 요건 (Acceptance Criteria)
  • 이전 완료된 Job ID와 피드백 문장을 받으면, 이전 코드를 신규 Job 공간으로 이전 복사하고 LLM에 컨텍스트로 함께 주입한다.
  • 수정 완료 시 갱신된 새로운 STL과 이미지가 버전업 형태로 다운로드 제공된다.

🛡️비즈니스 리스크 완화를 위한 보안 및 안정성 제약

LLM 기반으로 호스트 서버에서 프로세스를 대행 실행할 때 발생할 수 있는 보안 취약성 및 시스템 마비를 방지하기 위한 핵심 장치들입니다.

⚠️디렉토리 탈출 차단 (Traversal 방지)

LLM이 작업 계획을 세우면서 임의의 경로(예: `../../etc/passwd` 등)를 요청하더라도, 모든 파일 쓰기/폴더 생성 시 `Path.resolve()` 기반의 물리적 하위 디렉토리 영역 엄격 매핑 검증을 수행합니다. 위반 감지 시 즉시 Job을 중단하고 실패 처리하여 해킹 위협을 방지합니다.

🚫명령어 주입 차단 (No Shell Execution)

LLM이 쉘 커맨드를 임의로 조작하여 임의 명령어를 실행하지 못하도록 사전에 허용된 작업 스키마(WRITE, DIR_CREATE, RUN_TOOL)만 파싱하여 허용합니다. CLI를 실행할 때도 OS 쉘을 거치지 않는 프로세스 인자 주입 방식(`["openscad", ... ]`)을 고수해 Command Injection 가능성을 완벽히 제거합니다.

⏱️자원 고갈 예방 (OS 리소스 및 시간 제한)

사용자의 렌더링 코드에 실수로 무한 루프가 주입되어 서버 CPU/메모리가 고갈되는 것을 차단하기 위해, CLI 실행에 최대 30초의 타임아웃을 강제하여 초과 시 즉시 프로세스를 강제 종료(Kill)합니다. 또한, 악성 스크립트가 대용량 쓰기를 시도할 경우 파일 크기를 제한하여 호스트 디스크를 보존합니다.

🪣임시 작업 영역 정리 (Workspace Clean-up)

렌더링 수행 과정에서 발생하는 수많은 임시 파일들이 디스크에 영구 누적되어 용량 장해를 일으키는 것을 막기 위해, Job의 완료/실패 즉시 저장 완료된 최종 Artifact를 제외한 임시 작업 디렉토리 내의 찌꺼기 파일들을 완전 소거(Clean-up)합니다.

📅프로젝트 수행 로드맵

현재 기획(Inception) 단계를 완수하였으며, 향후 구현(CONSTRUCTION) 단계에서는 5가지 개발 단위(Unit of Work)에 맞추어 순차적으로 빌드, 테스트 및 검증하며 안전하게 릴리즈합니다.

Unit 1 개발 1단계

API Core & Storage Service

FastAPI API 뼈대를 구축하고 PostgreSQL 데이터베이스 연결 및 Job 테이블을 구현합니다. 또한 Workspace 디렉토리를 생성/정리하고 아티팩트를 보관할 추상 `StorageService`와 호스트용 로컬 구현체를 완성합니다.

Unit 2 개발 2단계

LLM Plan Parser & Validator

LLM이 응답한 JSON Action Plan을 안전하게 로드하고, Pydantic으로 스키마 규격을 유효화합니다. 경로 traversal 우회 해킹 시도를 탐지하여 차단하는 `SecurityPolicyValidator`와 유효성 실패 검증 단위 테스트를 개발합니다.

Unit 3 개발 3단계

CLI Execution Runner

호스트 OS 상에서 격리된 상태로 OpenSCAD CLI를 실행할 `CLIExecutionRunner`를 개발합니다. 30초 타임아웃 강제 중단 기능 및 CLI 비동기 로그 수집 장치를 완비합니다.

Unit 4 개발 4단계

SSE Streaming & Event Catch-up

실시간 진행 상황을 중계하는 SSE 스트림 엔드포인트를 열고, DB 로그 테이블을 0.5초 간격으로 스캔하는 SQL Polling 스레드를 구축합니다. 클라이언트 재연결 시 `Last-Event-ID`를 매핑해 누락 이벤트 로그를 복구 전송하는 기능을 완료합니다.

Unit 5 개발 5단계 (최종 통합)

Iterative Refinement Orchestrator

상세 유닛들의 기능을 결합하는 `JobOrchestratorService`를 완성하여 전체 비동기 라이프사이클을 매끄럽게 연결합니다. 이전 성공했던 Job의 코드를 자동 이전 복사하여 대화형 추가 수정 피드백을 반영하는 점진적 모델 개선 시나리오를 완성합니다.
