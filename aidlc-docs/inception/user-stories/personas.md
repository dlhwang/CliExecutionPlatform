# 페르소나 정의서 (User Personas)

본 문서는 **LLM 기반 Workspace CLI Execution Platform**의 핵심 타겟 사용자 페르소나를 정의합니다.

---

## Persona: 김민수 (Minsu Kim) - 일반 사용자 & 3D 프린팅 취미가

```
+-------------------------------------------------------------+
| 이름: 김민수 (32세)                                         |
| 역할: 3D 프린팅 입문자 및 메이커 취미가                     |
| 전문 기술: CAD/OpenSCAD 코딩 능력 없음, 간단한 슬라이서 사용 |
+-------------------------------------------------------------+
```

### 1. 배경 및 프로필
- 민수는 최근 가정용 3D 프린터를 구매하여 일상생활에 필요한 소품(예: 책상 정리함, 스마트폰 거치대, 스마트워치 충전 도크)을 직접 출력하는 재미에 빠져 있습니다.
- Thingiverse나 Printables에서 기존 모델을 다운로드받아 출력해 보았으나, 본인이 가진 스마트워치(샤오미 워치 S4) 크기에 정확히 맞는 충전 도크 디자인을 찾지 못해 직접 설계하고 싶어 합니다.
- 하지만 전문 3D CAD 도구(Fusion 360, Tinkercad 등)나 코딩 기반의 OpenSCAD를 배우기에는 학습 곡선이 너무 높고 시간적 여유가 없습니다.

### 2. 플랫폼 사용 목적
- 자연어로 원하는 형태를 설명하여(예: "샤오미 워치 S4용 가로 거치형 충전 도킹스테이션 설계도 만들어줘") 동작 가능한 OpenSCAD 코드(.scad)와 3D 모델링 결과물을 얻고자 합니다.
- 시스템이 디자인을 생성하고 렌더링하는 실시간 과정을 브라우저나 클라이언트 화면에서 투명하게 확인(로그 및 상태 스트리밍)하고 싶어 합니다.
- 완성된 3D 렌더링 이미지(preview.png)를 보고 형상이 마음에 들면 최종 3D 프린팅용 파일(output.stl)을 다운로드받아 자신의 슬라이싱 소프트웨어로 넘기고자 합니다.
- 렌더링 결과를 본 후 크기가 조금 작거나 디테일을 수정하고 싶을 때, 대화하듯이 추가 요구사항(예: "충전기 끼우는 구멍 지름을 2mm만 더 키워줘")을 입력하여 설계를 반복적으로 개선하고 싶습니다.

### 3. 주요 Pain Points (해결하고자 하는 불편함)
- **높은 CAD 진입장벽**: 3D 모델링 설계에 너무 많은 노력과 시간이 소요됨.
- **불투명한 처리 과정**: 요청을 보낸 후 서버에서 무엇을 하고 있는지 알 수 없어 답답함. (중간에 멈춘 건지, 렌더링 중인지에 대한 피드백 부족)
- **네트워크 단절 불안**: 스트리밍 로그를 보던 중 브라우저를 새로고침하거나 인터넷이 잠깐 끊겼을 때, 진행 중인 내역과 로그가 날아가 버려 처음부터 다시 요청해야 하는 불편함.
- **수정의 어려움**: 한 번 생성된 결과물에서 아주 작은 치수만 바꾸고 싶어도, 기존 CAD 코드를 직접 수정할 줄 몰라 처음부터 새로 생성해야 하는 비효율성.

### 4. 핵심 시나리오
1. **자연어 요청**: "샤오미 워치 S4 충전 도킹스테이션 만들어줘" 라고 입력 후 제출.
2. **실시간 모니터링**: 서버가 Job을 생성하고 LLM이 스크립트를 작성하며 OpenSCAD로 모델을 렌더링하는 모든 로그가 실시간(SSE)으로 흐르는 화면을 시청.
3. **무중단 복구 경험**: 모니터링 중 실수로 브라우저 새로고침을 누르거나 네트워크가 끊겨 재연결되었을 때, 누락된 로그를 끊김 없이 다시 전송받아 마저 모니터링함.
4. **결과물 확인**: 생성된 `preview.png` 이미지를 보고 전체적인 외관 확인.
5. **반복 피드백 수정**: "오른쪽 벽면에 케이블 통과용 홈을 추가해줘"라고 추가 입력을 통해 도면 수정.
6. **최종 산출물 다운로드**: 최종 `output.stl` 파일과 OpenSCAD 코드(`model.scad`)를 다운로드하여 3D 프린터로 출력.

## R-16 Artifact ID 기반 보안 다운로드 역할 확장

### 주요 페르소나 R16-P1: 김민수 - 최종 사용자

#### 목표
- 다운로드 링크에 서버 경로나 파일명을 직접 입력하지 않고 생성된 결과물을 안전하게 내려받는다.
- 결과 파일이 원래 이름과 올바른 미디어 타입으로 제공되어 브라우저 또는 후속 도구에서 정상적으로 처리되기를 기대한다.
- 사용할 수 없거나 차단된 결과물에 대해 서버 내부 경로가 노출되지 않는 일관된 오류를 받는다.

#### R-16 사용자 여정
1. 완료된 Job 결과 화면에서 서버가 제공한 `artifact_id` 기반 다운로드 링크를 선택한다.
2. 클라이언트는 `GET /api/v1/artifacts/{artifact_id}/download`를 호출한다.
3. 정상 Artifact이면 저장된 파일명과 미디어 타입으로 파일을 내려받는다.
4. Artifact가 없거나 물리 파일이 사라진 경우 안전한 404 오류를 확인한다.
5. 변조된 경로가 탐지된 경우 서버 경로가 없는 403 오류를 확인한다.

### 보조 페르소나 R16-P2: API 클라이언트 개발자

#### 목표
- 경로나 파일명을 조합하지 않고 `artifact_id` 하나로 다운로드 기능을 연결한다.
- HTTP 403과 HTTP 404를 구분하고 `Content-Type`, `Content-Disposition`을 신뢰할 수 있는 계약으로 사용한다.

### 보조 역할 R16-A1: Artifact 등록 주체

#### 설명
오케스트레이션 실행 중 생성된 파일을 Artifact로 승격하고 DB 메타데이터를 등록하는 서버 내부 컴포넌트다. 사용자 페르소나는 아니지만 R-16 여정의 선행 보안 경계를 소유한다.

#### 책임
- workspace root 기준의 안전한 상대경로만 등록한다.
- 절대경로, 빈 경로, `.`, `..`, `../` segment가 포함된 경로는 메타데이터 영속화 전에 거부한다.
